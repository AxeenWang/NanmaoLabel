## 使用說明

- 每條 checklist 皆可追溯至 raw_spec.md 具體條款
- Pass Criteria 為驗收通過的最低標準
- 驗證方式說明如何執行驗收
- `[ ]` 表示待驗收，`[x]` 表示通過，`[!]` 表示失敗

## 1. 核心驗收三要素 [ref: raw_spec 10, 13.Ω]

> ⚠️ **三項必須同時成立，方為驗收通過**

### 1.1 可掃描性 [ref: raw_spec 13.24]

| ID | 檢核項目 | Pass Criteria | 驗證方式 | 狀態 |
|----|----------|---------------|----------|------|
| AC-001 | Code 128 條碼首次掃描成功 | 手機/掃描槍首次嘗試即成功讀取 | 列印 PDF，使用至少 2 種設備掃描 | [ ] |
| AC-002 | QR Code 首次掃描成功 | 手機/掃描槍首次嘗試即成功讀取 | 列印 PDF，使用至少 2 種設備掃描 | [ ] |
| AC-003 | 無需特定角度或距離 | 一般手持距離 (10-30cm) 即可成功 | 多角度、多距離測試 | [ ] |
| AC-004 | 不同設備成功率一致 | 主流手機相機與掃描槍皆可成功 | 至少 2 種設備交叉驗證 | [ ] |

**違規判定** [ref: raw_spec 13.24]：
- S-01: 需多次嘗試 → ❌ 不穩定 → 驗收不通過
- S-02: 需特定角度/距離 → ❌ 不穩定 → 驗收不通過
- S-03: 不同設備成功率不一致 → ❌ 不穩定 → 驗收不通過

### 1.2 內容正確性 [ref: raw_spec 13.25]

| ID | 檢核項目 | Pass Criteria | 驗證方式 | 狀態 |

| AC-006 | QR Code 內容與資料一致 | 掃描結果 = 組合欄位 Raw Value（分號分隔） | 掃描後比對 data.json | [ ] |

| AC-008 | QR Code 空值保留位置 | 缺值欄位輸出 `A;;C` 格式 | 測試含空值欄位的 QR Code | [ ] |
